# Logic Workflow (Based on Flowchart)
请严格遵守以下执行顺序：

1. **检查服务状态 (Check Service)**
   - 检查 `hysteria-server.service` 是否存在或正在运行。
   - **分支 A (存在)**：
     - 执行 `systemctl stop hysteria-server.service` 停止服务。
     - 调用官方卸载命令：`bash <(curl -fsSL https://get.hy2.sh/) --remove`。
     - 输出提示：“服务已移除”。
   - **分支 B (不存在)**：
     - 输出提示：“未检测到 Hysteria 服务安装，跳过服务卸载步骤”。
   - *无论走哪个分支，完成后均进入下一步。*

2. **检查防火墙规则 (Check IPTables NAT)**
   - 检查 `iptables -t nat -L PREROUTING` 中是否有规则。
   - **分支 A (有规则)**：
     - **列出规则**：读取并打印当前的 NAT PREROUTING 规则列表（需显示行号，方便选择）。
     - **用户交互 (User Input)**：
       - 提示用户：“请输入要删除的规则行号（留空回车则默认不删除）：”
       - *逻辑要求*：这是本脚本唯一的交互点。如果用户输入数字，则执行 `iptables -t nat -D PREROUTING [行号]` 删除对应规则；如果用户直接回车，则保留规则。
     - **持久化**：如果执行了删除操作，需运行 `netfilter-persistent save` 保存更改。
   - **分支 B (无规则)**：
     - 直接跳过，不进行任何操作。

3. **结束程序 (Exit)**
   - 提示“环境清理/卸载流程结束”，退出脚本。
